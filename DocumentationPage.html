Basic Tags

  • <h1></h1> are heading tags, they start big and go small down to h6
  • <p></p> body paragraph
  • <!-- are comments -->

HTML5 Tags

  • HTML5 introduces more descriptive HTML tags: Main, header, footer, nav, video, article, section, and others
  • Helps with SEO (Search engine optimization)
  • Main helps find the main content of your page
    • <main>
      <h1>Hello World</h1>
      <p>Hello Paragraph<p>
    </main>

Images

  • Add images with: <img src="url">
  • They should all have an alt attribute incase the image doesn’t show, to describe what the image is, good for the visually impaired
    • <img src=”url” alt="Image description">

Anchor Tag

  • Use a (anchor) elements to link content outside of your webpage
  • Need a destination called href attribute
  • Eg. <a href=”url”>link description</a>
  • Use anchor elements to also create internal links to jumps to different sections within a website
  • Use # with the value of id attribute for the element
    • <a href="#contacts-header">Contacts</a>
    ...
    <h2 id="contacts-header">Contacts</h2>
  • target=”_blank” causes it to open in another tab
    • <a href="#contacts-header" target=”_blank”>Contacts</a*gt
  • Dead links just with href=#

Lists

  • <ul> for unordered lists, <ol> for ordered (ordered is numbered)
  • <li> is a list statement
    • <ul>
      <li>milk</li>
      <li>cheese</li>
    </ul>

Text Fields

  • Input let’s you get input from user
  • Eg. <input type=”text”>
  • Add placeholder text in input element with Eg. <input type=”text placeholder=”text”>
  • Create form element to send data to server using form action
  • <form action="/url-where-you-want-to-submit-form-data"></form>

Buttons

Submit Button

  • <button type="submit"> this button submits the form</button>
  • Goes inside the form element
  • You can require fields before form is allowed to be sent
  • Add attribute required
  • <input type=”text” required>
  • When a form gets submitted the data is sent to the server including all the options set by the buttons
  • The values are reported from the value attribute
  • If you omit the value attribute then the default will be name=on
    • <label for="indoor">
      <input id="indoor" value="indoor" type="radio" name="indoor-outdoor">Indoor
    </label>
    <label for="outdoor">
      <input id="outdoor" value="outdoor" type="radio" name="indoor-outdoor">Outdoor
    </label>

Radio Button

  • Each button can be nested between a label element
  • It will be associated with the label element surrounding it
  • All related radio buttons should have the same name attribute to create a group
  • A group let’s you select only one radio button from them
  • Best practice to set a for attribute on the label element, with a value of the id attribute of the input element
  • Let’s assistive technology create a linked relationship between the label and the child input
  • Usually inside form
    • <label for="indoor">
      <input id="indoor" value="indoor" type="radio" name="indoor-outdoor">Indoor
    </label>

Checkboxes

  • Same as radio except type=”checkbox”

Div

  • Div element is a division element, a container for other elements

Doctype

  • Need to tell the browser which version of HTML your page is using
  • And need to wrap entire html document in <html></html> tags
  • You can add other html tags for more order
  • Markup info goes into head tag, content goes into body tag
  • Metadata elements (link, meta, title, and style) typically go inside the head element
    • <!DOCTYPE html>
    <html>
      <head>
        <!-- metadata elements -->
      </head>
      <body>
        <!-- page contents -->
      </body>
    </html>>
